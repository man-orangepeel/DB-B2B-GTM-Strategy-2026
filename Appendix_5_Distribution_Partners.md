## 1. Méthodologie — scoring canaux de distribution (6 critères pondérés)

Grille appliquée aux OF candidats susceptibles de distribuer la formation DB à leur portefeuille pros :

| # | Critère | Poids | Évalue | Échelle 0-5 |
|:---:|---|:---:|---|---|
| **C1** | **Adéquation audience** | 20 % | Alignement segments DB (notaires + EC libéraux + EC salariés IDCC 787) | 0 = aucun chevauchement · 5 = 100 % audience = cible DB |
| **C2** | **Volume distributif** | 20 % | Taille portefeuille pros / nombre formés/an potentiel via le canal | 0 = < 50 pros/an · 5 = > 5 000 pros/an |
| **C3** | **Accessibilité relationnelle** | 20 % | Relai réseau DB existant vs approche cold · probabilité conversion contact→accord | 0 = cold absolu · 5 = relation DB directe préétablie |
| **C4** | **Maturité partenaire** | 15 % | Qualiopi + stabilité + ancienneté + offre crypto/blockchain existante | 0 = non-Qualiopi + instable · 5 = Qualiopi + leader + offre crypto documentée |
| **C5** | **Rentabilité attendue** | 15 % | Modèle rétrocession (15-30 % vs 70-85 % DB) · volume session × récurrence | 0 = one-shot marge faible · 5 = partenariat durable marge 85 %+ |
| **C6** | **Risque stratégique** | 10 % | Concurrence interne + exclusivité demandée + dépendance (facteur correctif) | 0 = concurrent direct + exclusivité · 5 = zéro concurrent + non-exclusif |

**Score total /100** · Seuils interprétatifs : ≥ 70 = **P0** · 55-69 = **P1** · 40-54 = **P2** · 25-39 = **P3** · < 25 = **Hors scope**.
